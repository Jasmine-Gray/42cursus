### 1. `std::cout` 

* **`std::` (名前空間 / Namespace)**
    「スタンダード（標準）」の略です。C++が最初から用意してくれている便利な機能は、他の自作プログラムと名前が被らないようにすべて `std` というグループ（名前空間）の中に閉じ込められています。
* **`cout` (シーアウト)**
    **C**haracter **OUT**put（文字出力）の略です。`<iostream>` というヘッダーファイルをインクルードすると使えるようになる「出力ストリーム（データの通り道）」のオブジェクト（部品）です。
* **`<<` (挿入演算子 / Insertion Operator)**
    データを `cout` に流し込むための矢印（パイプ）だと考えてください。右側のデータを、左側の `cout`（出力ストリーム）に向かって「ドンッ！」と押し込むイメージです。

```cpp
int age = 42;
std::string name = "Alice";
std::cout << "Name: " << name << ", Age: " << age << std::endl;
```
C++の `std::cout` は**「型を自動で判別」**してくれます！
文字だろうが、整数だろうが、小数点だろうが、とにかく `<<` で繋ぐだけで、コンパイラが裏で勝手に適切な形に変換して出力してくれます。これを**「型安全（Type Safety）」**と呼び、C++の非常に強力な強みの一つです。

### 2. `std::endl`

`std::cout` の最後によく付いてくる `std::endl` は、**End Line（行の終わり）**の略です。
これには重要な2つの役割があります。

1.  **改行（`\n`）を出力する**
2.  **バッファをフラッシュする（Flush）**

コンピューターは効率化のために、文字をある程度溜め込んでから一気に画面に描画します（バッファリング）。`std::endl` を使うと、「今すぐ溜まっている文字を画面に出力して！」と強制的に押し出す（フラッシュする）ことができます。

※ちなみに、単純に改行したいだけでフラッシュが不要な場合は `std::cout << "Hello\n";` のように `\n` を使う方が、処理速度はほんの少し速くなります。

## unsigned char キャストする理由 
「コンパイラの環境や入力される文字によっては、プログラムがクラッシュする（未定義動作を起こす）のを完全に防ぐため」**です。
きゃすと
なぜそんなことが起きるのか、その裏側にあるC++（およびC言語）の少し厄介な仕様を、3つのステップで分かりやすく解説します。

1. std::toupper の隠された仕様
std::toupper（や isalpha, isdigit など）は、文字を大文字に変換する関数ですが、実は引数として char ではなく int（整数） を受け取るように作られています。

そして、C++の標準規格（ルールブック）には、この関数に渡していい数字の範囲が厳密に定められています。

ルール： 引数は必ず EOF（通常は -1）、または **unsigned char で表現できる範囲（0 〜 255）**の数字でなければならない。

2. char 型の裏切り（マイナスになる可能性）
私たちが普段使っている char 型ですが、実は環境（コンパイラやOS）によって「符号付き（マイナスの値を持てる）」か「符号なし（0以上のみ）」かが変わります。そして、MacやLinuxを含む多くの環境では char は「符号付き（Signed）」として扱われます（範囲：-128 〜 127）。

アルファベットの 'A'（65）や 'z'（122）のような通常のASCII文字ならプラスの数字なので全く問題ありません。

問題は、ユーザーが想定外の文字を入力した時です。
例えば、ヨーロッパ言語のアクセント記号付きの文字（'é' など）や、日本語（全角文字）の一部などが入力された場合、コンピュータはその文字を**マイナスの数字（例：-61）**として読み取ってしまうことがあります。

3. マイナスの値を渡すとどうなるか？（大惨事）
もし、マイナスの値（例：-61）を持った char をそのまま std::toupper に渡してしまうと、ルール違反となります。

昔のC言語のライブラリでは、toupper を「配列のインデックス（住所）」を使って高速に処理するように実装されていることがよくありました。
変換結果 = 変換テーブル[入力された文字の数字];

ここにマイナスの数字を渡してしまうと、**「配列のマイナス番地（存在しないメモリ領域）にアクセスしろ」**という命令になってしまいます。結果として、セグフォ（Segmentation fault）でプログラムが強制終了したり、全く関係ないメモリのゴミデータを返してきたりします。これを「未定義動作（Undefined Behavior = UB）」と呼びます。

## ex01: My Awesome PhoneBook

**テーマ：「オブジェクト指向プログラミング（OOP）の基礎と堅牢な入力処理」**

#### 1. クラス設計とカプセル化（Encapsulation）

* **インスタンス化**: `PhoneBook` と `Contact` という「設計図（クラス）」を作り、`main` 関数やクラスの内部で「実体（インスタンス）」として生成する。動的確保（`new`）を使わず、静的配列としてメモリを確保する。
* **アクセス修飾子によるデータ隠蔽**:
* `private`: 連絡先の名前や電話番号など、外部から直接書き換えられては困るデータを隠す。
* `public`: `add_contact` や `get_first_name` など、外部との安全な窓口となる関数（メソッド）を公開する。


* **Getter と Setter**: `private` なデータを安全に読み書きするための専用の関数（例：`set_info`, `get_first_name`）。Getterには `const` を付け、「この関数は絶対にデータを書き換えない」という誓約を持たせる。

#### 2. 安全な標準入力（Standard Input）

* **`std::getline()` の必須化**: `std::cin >>` では空白文字（スペース）で入力が途切れてしまうため、1行を丸ごと取得する `std::getline` を使用する。
* **EOF（Ctrl+D）対策**: `std::getline` の戻り値（または状態）を確認し、入力ストリームが強制切断された場合に無限ループに陥らないよう、安全に `break` や `return` する処理が不可欠。
* **入力のバリデーション（Validation）**: `.empty()` で空入力を弾く、`std::isdigit` で電話番号が数字のみかチェックするなど、「ユーザーは間違った入力をする」という前提で堅牢なコードを書く。

#### 3. アルゴリズムとフォーマット

* **リングバッファ（環状バッファ）**: 9件目の追加時に最も古いデータを上書きするロジック。`this->_contact_count % 8` のように、モジュロ演算（余り）を使うことで、`if` 文を使わずに自動で0〜7のインデックスをループさせる賢い手法。
* **文字から数値への変換**: `SEARCH` 時にユーザーが入力した文字（例：`'3'`）を、ASCIIコードの特性を利用して `input[0] - '0'` と計算し、実際の数値（`3`）に変換するテクニック。
* **`<iomanip>` による出力制御**: 一覧表示における `std::setw(10)`（10文字幅指定・右揃え）の使用と、10文字を超える場合の `.substr()` による文字列の切り出し（切り詰め処理）。


## アクセサー, ゲッター, セッター
「アクセサー」という大きなくくりの中に、「ゲッター」と「セッター」の2種類が含まれているという関係性。


1. アクセサー (Accessor)
意味: クラスの内部にある「隠されたデータ（プライベート変数）」にアクセス（参照・変更）するためのメソッド（関数）の総称です。

役割: オブジェクト指向の重要概念である「カプセル化（データを外部から勝手にいじられないように保護すること）」を実現するための窓口です。

2. ゲッター (Getter)
意味: アクセサーの一種で、データを**「取得する（読み取る）」**ためだけのメソッドです。

特徴: * 一般的に get変数名() という名前が付けられます（例: getName(), getHp()）。

クラスの内部データを返すだけで、データを書き換えることは絶対にしません。

C++などでは、データを変更しないことを保証するために関数の末尾に const をつけるのがお作法です。

3. セッター (Setter)
意味: アクセサーの一種で、データを**「設定する（書き換える）」**ためのメソッドです。

特徴: * 一般的に set変数名(引数) という名前が付けられます（例: setName("Taro"), setHp(100)）。

単に値を代入するだけでなく、**「不正な値が入力されないかチェックする（バリデーション）」**という非常に重要な役割を持ちます。
