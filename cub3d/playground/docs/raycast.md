⏺ レイキャスティング詳解

  Lode's Tutorial の内容を日本語で解説します。

  1. 基本原理：なぜ高速なのか

  【通常の3D】                    【レイキャスティング】
  全ピクセルを計算                 縦線だけ計算
    640 x 480 = 307,200回           640回だけ！

  ポイント: 画面の各「縦の列」に対して1本のレイ（光線）を飛ばすだけ。

  ---

  2. プレイヤーの表現（ベクトル）

          planeX, planeY（カメラ平面）
           ←─────────────→
                │
                │ dirX, dirY（方向ベクトル）
                │
                ▼

           ◉ posX, posY（プレイヤー位置）

  3つのベクトルで表現：

- pos (posX, posY) - プレイヤーの位置
- dir (dirX, dirY) - 見ている方向
- plane (planeX, planeY) - カメラ平面（視野の幅）

  視野角 (FOV):

- dir と plane の長さの比率で決まる
- 同じ長さ = 90度
- dir が長い = 狭い視野（ズームイン）

  ---

  3. レイの計算

  画面の x 座標（0 ~ 画面幅）ごとに：

  // cameraX: -1.0（左端）〜 +1.0（右端）
  double cameraX = 2 * x / (double)screenWidth - 1;

  // レイの方向 = 向いている方向 + カメラ平面 × cameraX
  double rayDirX = dirX + planeX *cameraX;
  double rayDirY = dirY + planeY* cameraX;

  図解：
                      画面
          ┌─────────────────────────┐
          │ x=0    x=320    x=639   │
          └─────────────────────────┘
              ↖      ↑      ↗
                ↖    │    ↗       各縦線に1本のレイ
                  ↖  │  ↗
                    ↖│↗
                     ◉ プレイヤー

  ---

  4. DDAアルゴリズム（壁検出の核心）

  なぜDDAが必要か：
  悪い方法：一定間隔でチェック      良い方法：グリッド境界でチェック
    ・・・・・・█                    →  →  →  █
    壁を飛び越す可能性あり！            確実に壁を検出

  DDAの仕組み：
  ┌───┬───┬───┬───┬───┐
  │   │   │   │ █ │ █ │  マップ（1マス = 1単位）
  ├───┼───┼───┼───┼───┤
  │   │   │ *│ █ │   │* = レイがグリッド線を横切る点
  ├───┼───┼*──┼───┼───┤
  │   │ * │   │   │   │
  ├───┼*──┼───┼───┼───┤
  │ ◉ │   │   │   │   │  ◉ = プレイヤー
  └───┴───┴───┴───┴───┘

  核心の変数：
  // 次のX/Y境界までの距離
  double sideDistX, sideDistY;

  // X/Y境界間の距離（レイの傾きに依存）
  double deltaDistX = fabs(1 / rayDirX);
  double deltaDistY = fabs(1 / rayDirY);

  // 進む方向（+1 or -1）
  int stepX, stepY;

  DDAループ：

  ```c
  while (hit == 0)
  {
      if (sideDistX < sideDistY)
      {
          sideDistX += deltaDistX;  // X方向に進む
          mapX += stepX;
          side = 0;  // 縦の壁に当たった
      }
      else
      {
          sideDistY += deltaDistY;  // Y方向に進む
          mapY += stepY;
          side = 1;  // 横の壁に当たった
      }
      if (map[mapX][mapY] > 0)
          hit = 1;  // 壁発見！
  }
  ```

  ---

  5. 壁の高さ計算（魚眼補正付き）

  重要：ユークリッド距離を使うと魚眼効果が出る！

  ❌ 悪い例：実際の距離を使う
     画面端の壁が膨らんで見える（魚眼レンズ効果）

  ✅ 正解：カメラ平面への垂直距離を使う
     自然な見え方になる

  正しい距離の計算：
  if (side == 0)  // 縦の壁
      perpWallDist = sideDistX - deltaDistX;
  else            // 横の壁
      perpWallDist = sideDistY - deltaDistY;

  // 壁の高さ
  int lineHeight = (int)(screenHeight / perpWallDist);
